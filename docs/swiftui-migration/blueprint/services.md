# services (`services`)

**Effort:** L

## 概述

The services layer holds Fresh Pantry's stateless/IO-boundary logic split into three groups: (1) AI + web integrations — an OpenAI-compatible chat client (ai_client) driving a text/vision ingredient parser and a URL->recipe parser, fed by a Safari-style recipe-page scraper, plus an Open Food Facts client for product details and per-100g nutrition; (2) device-service wrappers — local-notification scheduling (notification_service + pure expiry_scheduler), versioned JSON backup codec (backup_service), and DI-seam abstractions for inbound invite deep links (invite_link_service) and system/clipboard share intents (share_intent_service); (3) the pure Intake/Deduction domain engine — proposal_planner (fuzzy matching + ADR-0001 merge identity), intake/deduction proposal factories, and ingredient_factory that turn AI drafts, shopping items, and cooked recipes into reviewable inventory mutations. Most files are pure functions or thin actors over URLSession-equivalent HTTP, making them clean 1:1 Swift ports; the parity risk concentrates in deterministic id hashing, the merge-identity rule, and exact OFF/LLM JSON contracts. The Cloudflare invite API and token generation/hashing actually live in the sync/household subsystem — services only receive the resulting deep link.

## 组件(17)

### lib/services/_http.dart

_Shared HTTP GET helper: 8s timeout, 1 retry, 500ms backoff on TimeoutException/ClientException._

Single top-level function `Future<http.Response> fetchWithRetry(Uri uri, {http.Client? client, Duration timeout = 8s, Duration retryDelay = 500ms, int retryCount = 1, Map<String,String>? headers})`. Loop `for attempt in 0..retryCount`: do `client.get(uri, headers).timeout(timeout)`; on TimeoutException or ClientException rethrow only on last attempt, else `await Future.delayed(retryDelay)` and retry. If `client` is null it creates a transient http.Client and closes it in finally; if provided, caller owns lifecycle. Swift: an `actor`/struct method using URLSession with a configured `timeoutIntervalForRequest=8` and a manual retry loop (catch URLError.timedOut / network errors), passing custom headers. Used only by OpenFoodFactsService.

## 迁移注意

PARITY-CRITICAL INVARIANTS: (1) ExpiryScheduler._idFor rolling hash `hash = (hash*31 + code) & 0x7fffffff` over Dart codeUnits — use Swift `.utf16` (not .unicodeScalars) so notification ids stay identical/stable; collision-avoid with dailySummaryId=1; dailySummaryHour=9 (09:00 LOCAL). (2) Notification scheduledAt is built from LOCAL Y/M/D-offset at 09:00; skip if not strictly after now; dailySummary uses time-only recurring trigger. (3) ReminderSettings.enabledOffsetDays order is [7,3,1] (D7,D3,D1) — preserve. (4) IngredientIdentity merge rule (ADR-0001): perishable categories {果蔬生鲜,肉类海鲜,乳品蛋类} OR FoodKnowledge.isPerishableName -> ALWAYS new batch; non-perishable merges only when name(ci)+unit(trim)+storage match AND target quantity is numeric; mergeTargetId is a list INDEX as string, NOT a stable id — callers MUST re-resolve by identity at apply time (DeductionCandidate carries inventoryRowId/Name/Unit precisely for this). (5) fuzzyMatchInventoryRows: a length-1 inventory name must NOT match a longer recipe term (蛋 vs 蛋糕); reverse stays loose; candidates sorted by expiry ascending (nulls last). (6) DeductionProposalFactory._initialDeductAmount always returns a parseable >0 number (falls back to '1' on unit mismatch) — a deduction must never silently apply 0. (7) formatQuantity: whole->int string, else 2-decimal rounded (kills float artifacts like 1.2000000002). (8) BackupService version=2 ONLY (v1 rejected); strict decode-before-write so a bad import can't partially overwrite; addHistory round-trips as an opaque map; food-details cache intentionally excluded. (9) FoodDetails cacheVersion=5 hardcoded — must bump in lockstep with the repo's _foodDetailsCacheVersion when nutrition schema changes (v4 = pre-nutrition, stale). (10) OFF: exact nutriment keys energy-kcal_100g/proteins_100g/carbohydrates_100g/fat_100g; category-keyword map FIRST-match wins; product quality score formula is user-visible (best result selection). (12) ai_recipe_parser: difficulty clamped 1-5, cookingMinutes<=0 -> 30. (13) Supported recipe hosts gate (lanfanapp.com, xiachufang.com) is enforced in BOTH extractUrl and the fetcher; subdomains ok, lookalikes rejected.
